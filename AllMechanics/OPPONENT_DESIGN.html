對手設計：行為、AI、未來路線v0.8 — 對手免疫賽道加成

沙盒目前有 3 個對手，每個有不同個性與行為樣式。本文記錄目前實作的對手機制、AI 邏輯，以及未來預計大改的方向。對手是這個遊戲決策深度的主要來源——每個對手都應該迫使玩家用不同的策略。

→ 遊戲規則 → 機制細節 → 賽道設計 → 卡牌設計

⚠ 重大變動（v0.8）：對手免疫所有賽道加成（add / mult / speedLimit）——對手速度不再受順風 / 逆風 / 彎道加乘 / 限速影響。但賽道災害仍會懲罰對手：c6 油污 50% 滑開、c7 坑洞速度減半都照常生效。賽道對對手變成「中立風險」：不會幫忙、只會懲罰。
🔮 未來重大改動：對手 AI 預計在 v0.9–v1.0 之間進行整體重做。本文記錄的選道邏輯、策略名稱、行為冷卻機制都屬於暫時實作、會被重構。詳見 §6 未來路線。

§ 1  對手概覽

沙盒包含 3 個對手：禿鷹（A）、清道夫（B）、破風者（C）。每個對手有自己的速度起始、追擊速度（當後車時用）、專注值，以及一組行為（behaviors）。

id名稱起始速度追擊速度專注值個性標語
A禿鷹60500脫逃型——避開玩家道、拉開距離
B清道夫60601戰術型——所在道讓雙方失加成、抵消尾流
C破風者50500獨行型——自顧自地跑最快路線、邊跑邊加速

速度欄位：

  • speed：對手成為當前對手（你前面那台）時的起始速度
  • chaserSpeed：對手成為追擊者（你後面那台、可能威脅你 PASS）時的速度
  • focus：專注值，玩家必須磨光才能真正甩開（即使超車 QTE 成功也不立刻名次提升）

§ 2  對手速度結算 v0.8 變動

當前規則：對手免疫加成、但仍受災害影響

對手速度有兩類來源：

  1. 對手自己的動作（boost / moveTo+boostAfter / stay+absBonus）— 主動加速
  2. 賽道災害（c6 油污滑開、c7 坑洞減速）— 被動懲罰

除此之外，對手不會因為踩到加成道而改變速度（順風 / 逆風 / 彎道 mult / speedLimit 對對手都無效）。

事件對手速度變化
boost amount: N+N
moveTo + boostAfter: N+N
stay + absBonus: true（B 清道夫強招）取對手所在道的 abs(add)，永遠正值
其他切道動作（無 boostAfter）不變
進入彎道 / 限速段 / 加成道不變（對手免疫加成）
c6 油污中道（forceCornerQte）50% 機率失控、滑到隨機鄰道（清道夫光環免疫）
c7 坑洞道速度 ÷ 2（清道夫光環免疫）

實作對應

// applyOpponentBonus（簡化）— 加成歸零
function applyOpponentBonus(currentSpeed, laneIdx, bypassAura = false) {
  const b = getLaneBonusFor(laneIdx, "opponent", bypassAura);
  if (!b) return currentSpeed;
  // 對手版 speedLimit 預設 null → 進不了彎道分支
  if (seg?.type === "bend" && b.speedLimit != null) {
    if (currentSpeed > b.speedLimit) return b.speedLimit;
    return currentSpeed;
  }
  // 對手版 add 預設 0 → 一般段也無變化
  return currentSpeed + (b.add ?? 0);
}

// handleOpponentForceQte（油污 c6 中道）— 災害仍生效（清道夫光環免疫）
function handleOpponentForceQte() {
  if (isOpponentAuraActive()) return;  // 清道夫光環免疫
  const b = getLaneBonusFor(app.opponentLane, "opponent");
  if (!b?.forceCornerQte) return;
  // 50% 失敗 → 對手滑到鄰道
  if (Math.random() < 0.5) { /* 滑開邏輯 */ }
}

// checkPotholeDamage（坑洞 c7）— 災害仍生效（清道夫免疫）
//   不走 getLaneBonusFor、直接讀 potholeLanes 跟 opponentLane 比對
//   命中且非清道夫 → app.opponentSpeed = Math.floor(opponentSpeed / 2)

關鍵在於 getLaneBonusFor(lane, "opponent") 對對手把 add / mult / speedLimit 三個數值欄位歸零，但保留 forceCornerQte / slipOnQteFail 等災害旗標（透過 {...bonus} 展開保留），所以 handleOpponentForceQte 仍能讀到並觸發。坑洞則完全繞過 getLaneBonusFor、直接做位置比對。

賽道對對手「破例有效」的方式

若想讓特定賽道對對手仍有影響，可在 lane bonus 加入 forOpponent 子物件。範例：

{ lane:0, add:10, mult:1, label:"順風道 +10",
  forOpponent: { add: 5 }                    // 對手吃 +5、玩家吃 +10
},
{ lane:1, add:0, mult:1.25, speedLimit:75, qteDiff:"hard", label:"內彎 ×1.25",
  forOpponent: { speedLimit: 60 }            // 對手在這道受 60 速限壓制、不吃 mult
},

目前所有賽道都未定義 forOpponent——對手對所有賽道完全免疫。這是預設行為，未來可逐步加回部份賽道對對手的影響。

設計意圖：原本對手吃賽道加成會造成「對手在彎道反而加速」、「賽道機制反幫對手」等違反直覺的場景。v0.8 把加成的好處完全屬於玩家——你選道是為了自己跑快——但災害仍對雙方有效，讓賽道機制不會變成玩家的「絕對工具」：對手雖然不被加成幫忙、卻也會在坑洞跟油污犯錯。賽道對對手變成「中立風險」、不會幫忙也不會偏袒。

§ 3  Behavior 系統（冷卻制）

每個對手有一組 behaviors：每個 behavior 帶 cooldown、weight、action 等參數。每次玩家動作末尾，引擎評估哪些 behavior cooldown 滿了（含「下一動就能觸發」的 1 動冷卻）、依 weight 優先級觸發。

Action 類型

action關鍵參數說明
moveTotarget: lane index 或 "playerLane"對手切到指定賽道。"playerLane" 切到「玩家上一個動作開始前所在的道」（玩家換道可閃過）
moveSmartstrategy: bestForSelf / avoidPlayer / dynamicAvoidOrBlock後手最佳化選道，AI 根據策略 + 賽道狀況評估
moveAdjacent—隨機切相鄰道
boostamount: 加速量對手速度永久 + amount
boostAfter附加參數（接在 moveTo / moveSmart 後）移動完直接 + boostAfter
bypassAuramoveSmart 的附加參數對手無視自己的光環（B 清道夫專用），可吃自己所在道的加成
absBonusmoveTo 的附加參數（B 強招用）本動賽道結算取 abs(add) 當加成、永遠拿正加成。需配合 bypassAura

moveSmart 三策略

這三種策略決定 AI 怎麼選道：

  • bestForSelf：選對自己速度最高的道，無視玩家位置。例：破風者（C）的兩個 behavior 都用這個。
  • avoidPlayer：選速度最高的道、但排除玩家動作前所在道。讓玩家吃不到尾流。
  • dynamicAvoidOrBlock：動態策略——玩家還沒吃尾流 → 等同 avoidPlayer；玩家已吃尾流 → 切到玩家動作前道（block）。例：禿鷹（A）的兩個 behavior。

避坑邏輯：所有 moveSmart 策略都會排除有坑洞的道（c7 賽段）。即使是 dynamicAvoidOrBlock 想 block 玩家、若玩家在坑洞道、AI 會切回 avoidPlayer 邏輯（保命優先於戰術）。

冷卻系統

每個 behavior 有 cooldown（CD），記錄上次觸發時的玩家動作數。當 當前動作數 − 上次觸發動作數 ≥ cooldown 時、CD 滿、可再次觸發。

  • 引擎逐一檢查所有 behavior、計算剩餘 CD（remaining）
  • 選 remaining 最小的（最快會觸發的）
  • 若多個同時 ready、依 weight 優先：strong > medium > weak
  • 觸發後重置該 behavior 的計時器

§ 4  各對手詳細行為

禿鷹（A）—— 脫逃型

idcooldownweightactionstrategyboostAfter
a-weak1weakmoveSmartdynamicAvoidOrBlock—
a-strong3strongmoveSmartdynamicAvoidOrBlock+20

每動都會出弱招（CD=1），每 3 動會出強招（CD=3、+20 速）。AI 動態判斷尾流狀態：若玩家在追尾流就跑、若玩家已吃 → 切到玩家道把尾流還給對手。

玩家對策：禿鷹強招的速度爆發很關鍵，玩家在強招出來前需要先累積足夠速度準備超車。換句話說：強招倒數 1 動時、必須準備好爆發。

清道夫（B）—— 戰術型

idcooldownweightactiontarget特殊參數
b-strong2strongmoveToplayerLanebypassAura + absBonus

B 沒有弱招——他的常駐威脅是被動光環：B 所在的道對雙方都失加成、且抵消尾流 +30。強招 2 動觸發一次：切到玩家動作前道、自己豁免光環，並用 absBonus（永遠拿正加成）爆發。

光環機制細節：

  • 常駐：B 所在道的 lane bonus 對「玩家」與「對手自己」都被清除（add=0, mult=1）
  • 玩家跟 B 同道 → 不吃尾流 +30（光環抵消）
  • 免疫所有賽道災害：清道夫站任何坑洞道、坑洞自動抵消（不降速）；站油污道、不會 50% 滑開。預告也因此維持原意圖 ⛔ 不降級成 ❗
  • v0.9+ 玩家也完全免疫災害：對手是清道夫時、玩家不管站哪道都不會被坑洞 / 油污 QTE 觸發。之前要「跟到清道夫所在道」才安全、現在整段都安全。checkPotholeDamage 中 playerSafeAura = playerHit && auraActive、checkBendSpeedLimit 對 forceCornerQte 也加判定。
  • bypassAura: true：B 在那一動無視自己的光環，可以吃 lane bonus（強招那一動）
  • absBonus: true：bypassAura 那一動套用「絕對加成」——若該道 add=-10（逆風）→ 對手反而吃 +10

玩家對策：B 的弱點是強招前的 2 動空窗期——這時候 B 沒攻擊招、光環雖在但你可以提前到不同道。強招倒數時、不要待在 B 預測會切到的道。正面效應：遇到清道夫 + c6 油污 / c7 坑洞同時出現的關卡、玩家可以無視災害、把心力放在加速。

破風者（C）—— 獨行型

idcooldownweightactionstrategyboostAfter
c-weak1weakmoveSmartbestForSelf+5
c-strong2strongmoveSmartbestForSelf+10

C 完全無視玩家，每動都選自己最快的道、再加 5 或 10 速。雖然強招數值不如禿鷹，但因為 v0.8 對手不吃賽道加成、C 的 bestForSelf 策略實際失效（沒有「最快道」可選了）—— 詳見 §6。

玩家對策（當前實作）：C 是這三個對手中威脅最低的。專注度 0、超車一次就甩開、累積速度比較容易。

§ 5  UI 預告與意圖系統

玩家可以從對手車頭正上方的名條面板看到下一個對手動作的預告。預告分兩層：

  1. 倒數：「X 動後」——再過幾個玩家動作會觸發
  2. 意圖 icon：⛔=block（切到你道）、💨=flee（躲開你）、❓=unknown（策略本身不可預測）、❗=disrupted（賽道干擾、結果偏離原意圖）；可疊加 ⚡（含加速 / 特殊效果）

意圖判定降級規則

當實際結果跟原意圖偏離時、icon 會降級成 ❓ 或 ❗，避免誤導玩家。兩種狀態語意不同：

Icon狀態含義
❓unknown對手的策略本身就是動態的（看當下道路最快、隨機切道）。沒有明確意圖可預告。
❗disrupted對手原本想 block / flee，但賽道有坑洞或油污、AI 會繞開、結果偏離原意圖。

觸發條件

  • ❓ unknown：
    • moveSmart bestForSelf：選自己最快道、無視玩家
    • moveAdjacent：隨機切相鄰道
  • ❗ disrupted（原本有明確意圖、但被賽道干擾）：
    • 有坑洞的賽段（c7） + 任何 moveSmart：AI 選道空間被坑洞限制、可能被擠到玩家道或被迫離開玩家道
    • 有油污的賽段（c6） + 任何 moveSmart：AI 用期望值評估、可能選到油污道、50% 滑開實際落點不確定
    • moveTo target=playerLane + 玩家站坑洞：AI 會切到「離玩家最近的安全道」、不真的同道
    • moveTo target=N + N 是坑洞：AI 留在原道、moveTo 失效
    • moveTo target=playerLane + 玩家站油污：50% 觸發 QTE 後滑到鄰道

    例外：清道夫（B）光環免疫所有賽道災害——當前對手是清道夫時、上述條件全部失效、預告維持原意圖（⛔ / 💨）不降級成 ❗。清道夫被坑洞自動抵消、被油污也不會滑開、AI 也不需要避坑。

大數據預測車隊牌

裝備「大數據預測」車隊牌後、預告升級為 full mode：除了倒數跟 icon，還會顯示對手下一招的具體文字描述（「切到你道+加速」、「躲開你+豁免」等）。

§ 6  未來路線：AI 重做 v0.9–v1.0

本節內容是未來計畫、不是當前實作。現有 AI（冷卻 + behavior 列表 + 三策略）會被替換成更系統化的設計。下面是預計方向、細節仍會調整。

為什麼要重做

  • 策略名稱混亂：avoidPlayer / bestForSelf / dynamicAvoidOrBlock 三個策略，命名不一致（兩個動詞、一個複合詞），新對手很難套上去
  • bestForSelf 在 v0.8 後實際無效：對手不吃賽道加成、「最快道」不再存在，破風者（C）的策略變成「隨機切道」
  • 強弱招二分太僵：每個對手只有 1–2 個 behavior、招式組合空間小、無法做出真正不同的個性
  • 缺乏「情境感知」：AI 不知道自己快輸 / 快贏、不知道玩家手牌、不知道接下來的賽段
  • 難以擴展：要加新對手就要寫新 behavior、新策略、新光環邏輯，沒有共用底層

新 AI 方向（暫定）

方向 A：行為樹（Behavior Tree）

每個對手寫一棵決策樹，節點是「條件 + 動作」。例：

禿鷹：
  IF 玩家速度 > 自己速度 - 10:
    切到自己最快道
  ELSE IF 玩家同道:
    切走（破壞尾流）
  ELSE:
    待命累積冷卻

方向 B：意圖 + 動作分離

把「對手想做什麼」（意圖）跟「怎麼做」（動作）拆開。意圖是高層目標（守住名次 / 反超 / 騷擾玩家）、動作是執行方式（換道 / 加速 / 停留）。新對手只要設定意圖偏好權重。

方向 C：基於玩家狀態的 reactive AI

對手 AI 讀取玩家當前狀態（速度、手牌、輪胎、與終點距離）後決策、不再依賴固定 CD。例如「禿鷹看到玩家速度即將追平就主動加速」。

共同改進項

  • 對手「情緒 / 壓力」系統：對手快被超車 → 變激進 / 變保守（依個性）
  • 對手記憶：記住玩家近 N 動的模式、避免被玩家完全讀懂
  • 多階段強招：對手有多個「等級」的招式、每個對手 3–5 個 behavior 而非 1–2 個
  • 對手間互動：禿鷹被超後變成追擊者、會干擾玩家 PASS

什麼會保留

  • 專注值機制（focus 點數、磨光才能甩開）
  • 對手免疫賽道加成（v0.8 確立的核心設計）
  • 名條 + 預告 UI 框架（含 icon / 倒數 / full mode）
  • 個性標語 / 顏色 / 角色定位三人組（脫逃 / 戰術 / 獨行）

§ 7  Playtest 觀察項

v0.8 上線後需要在 playtest 時觀察的項目：

  1. 對手難度是否下修過頭：對手不吃加成後、累積速度比以前慢，超車太容易？
  2. 對手互換是否仍有壓力：禿鷹的脫逃感 / 清道夫的光環威脅 / 破風者的累積感、誰最容易被忽略？
  3. 名條預告是否清楚：玩家是否能區分 ⛔/💨/❓/❗ 四種狀態；❗（賽道干擾）的場景是否常見、玩家會不會在意
  4. 賽道對玩家是否仍有意義：對手免疫後、玩家對賽道的策略選擇是否還夠豐富
