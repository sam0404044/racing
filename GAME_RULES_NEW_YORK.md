## 遊戲概要

- **形式**：雙人對抗（你操控綠車，對手為紅車 NPC）的 **賽車指令卡牌遊戲**，含 **車輛部件**、**耗材輪胎** 與指令卡上的 **附贈動作**。
- **起跑位置**：綠車名次在後（跟車）、紅車領先；目標是在賽場階段把握超車時機超越對手。
- **一局長度（試玩版）**：同一賽段內最多 **3 個回合**；結算後本局結束（試玩未做多賽段連貫）。

---

## 介面上會看到的車況數值

| 項目 | 說明 |
|------|------|
| 名次 | 1 = 領先，2 = 落後（試玩為兩車位置互換邏輯） |
| 速度 / 基礎速度 | 速度受 **基礎速度（BS）** 倍率影響（見下方通則） |
| 車體操控（VH）、抓地力 | 影響結算時的 **SP** |
| 檔位 | 影響 BS 與速度上限 |
| 動力（Power） | 行動階段打出指令時消耗；每回合開始重置為上限 |
| SP | 結算用綜合指標（見「結算與超車」） |
| INT | **場上可蓋放的指令張數上限**（含「負擔」佔位） |
| 防守狀態 | 部分指令可開盾，抵銷對方的干擾類效果 |

**通則（畫面上亦有提示）**：任何效果「增加速度」若標為依 BS 結算，則 **實際速度增量 = 數值 × 當前基礎速度**。例如「速度 +1（×BS）」在目前 BS 下會加該倍率後的速度。

## 回合結構（五階段）

每回合依序為：

1. **計畫階段**  
   - 動力重置為上限。  
   - **抽牌**：每回合抽 **4** 張（手牌上限 **7**，超出則棄掉）。  
   - 若上一回合因輪胎受到「下回合少抽牌」懲罰，會在本階段扣除抽牌數。

2. **通訊階段**  
   - 將手牌拖曳到賽道 overlay 的 **左側或右側**（**面朝下**蓋牌）。  
   - **非資訊指令**（複雜／簡易）出牌上限（程式實作）：  
     - 複雜最多 **1** 張  
     - 簡易最多 **2** 張  
     - **合計最多 2 張**（複雜 + 簡易 ≤ 2）  
   - **資訊指令**：張數 **不限**，且 **不占用 INT**。  
   - 複雜／簡易蓋牌仍需遵守 **INT ≤ 7**（場上己方指令張數 + 負擔佔位）。  
   - 確認後無法撤回（試玩版）；對手 NPC 亦會依策略蓋牌。

3. **行動階段**  
   - **先手**：名次較後的一方先動（落後者先）。  
   - 輪到你時：點擊場上自己的蓋牌 **翻開並結算**；須支付該牌的 **動力消耗**。  
   - **複雜指令**結算後通常 **自動 Pass**（除非帶有 **「熟練」** 關鍵詞）。  
   - **附贈動作**（見下節）：結算後可能要求你操作部件、再蓋一張牌、或消耗輪胎；可視 UI 選擇跳過。  
   - **反應**類指令：**不可主動翻開**，只在符合描述／事件時觸發。

4. **賽場階段**  
   - 決定是否嘗試超車；若要超車，選 **左側或右側**施壓（與通訊／場上牌側對應）。  
   - 亦可選 **不超**，進入下一回合或觸發賽段結束（第 3 回合仍不超則進入結算）。

5. **結算階段**  
   - 計算雙方 SP、判定超車結果、重置場上牌與部分車況，並結束本局試玩流程。

## 附贈動作（Bonus）

當指令寫有「附贈」並在結算後觸發時：

| 附贈 | 玩家要做的事 |
|------|----------------|
| **操作車子** | 在左側 **車輛部件** 面板點選部件，執行該部件牌堆 **頂端元件** 的效果（部件為循環牌堆）。若本回合已被「增壓輸出」封鎖，則 **無法再選動力單元**。 |
| **再下令** | 從手牌選一張：資訊牌會立即生效；簡易／複雜則再蓋到左或右側（面朝下）。 |
| **消耗輪胎** | 從輪胎耗材堆頂 **放逐**一層；耗完或規則淘汰時 **退賽**。 |

若部件元件抽完（牌堆空），動力單元等視為 **報廢 → 淘汰**。

## 部件與輪胎（試玩版摘要）

- **動力單元**：元件多为加速；「增壓輸出」會在本回合 **封鎖**後續再操作動力單元。本回合透過動力單元獲得的速度增量會用作 **空力連動**（如「引導式氣箱」）。  
- **空力系統**：提升 VH 等；「碳纖維鼻錐」可讓「可變擴散器」在當回合額外獲得抓地力。  
- **變速箱**：透過「打檔」類元件調整檔位；檔位會改 **基礎速度** 與 **速度上限**。  
- **輪胎耗材**：由頂到底消耗；最底層帶有「下回合抽牌 -1」懲罰。附贈「消耗輪胎」或特定指令會消耗層數。

## 指令卡關鍵詞（Tooltip 與規則）

| 關鍵詞 | 效果 |
|--------|------|
| **熟練** | 複雜指令執行後 **不強制 Pass**。 |
| **負擔** | 該牌從場上移除後仍 **佔用 1 格 INT**，直到賽段結束。 |
| **反應** | **不可主動翻開**；滿足條件時由事件系統觸發。 |

## 結算與超車（程式邏輯）

### SP（穩定指標）

結算時：

\[
\text{SP} = \max\left(0,\ \mathrm{round}(0.5 \times \mathrm{VH} + 1 \times \mathrm{Grip} + 2 \times \mathrm{Speed})\right)
\]

- 若任一方 SP **> 8**，該方 **壓力值 +1**（試玩懲罰／節奏壓力）。

### 嘗試超車（僅當你仍在第 2 位、對手第 1 位）

1. 選定 **左或右**後，該側 **雙方蓋牌會全部翻開**（如有）。  
2. 計算該側各自指令上的 **壓力值總和**（`pressure` 加總）。  
3. **若你的壓力 ≥ 對手壓力**：視為「超車窗口打開」。  
   - 若此時 **你的 SP > 對手 SP**：**超車成功**，名次互換，判定為勝利類結局。  
   - 否則：**魯莽超車**結局（試玩訊息標示，仍無名次互換）。  
4. **若你的壓力 < 對手壓力**：超車失敗；若你當前速度仍高於對手，速度會 **被拉低至對手速度**。名次不變。

未選擇超車時：維持位置，依回合數進入下一回合或結束賽段。

### 賽段結束後

- 場上指令進棄牌堆；**VH／抓地力／速度／動力顯示等重置**，**檔位保留**。  
- 試玩版會直接進入 **遊戲結束**畫面並顯示結局類型（超越成功／魯莽／平局／退賽等）。

## 牌庫與建牌

- **玩家牌庫**：`cards.js` 的 `cardPool` **複製兩份**後洗牌。  
- **對手牌庫**：`npcCardPool` **複製三份**後洗牌。  
- 牌庫抽空後會 **重洗棄牌堆**補牌。
