設計理念

v0.6 的核心張力翻轉：

  • v0.5 與之前：速度難取得，超車是門檻——玩家會被困在「不夠快」的挫敗迴圈。
  • v0.6 起：速度便宜，但有些事情會要你規劃降速（彎道限速 QTE、超車後失誤牌污染、輪胎磨損）。決策重點從「我夠不夠」轉到「我該不該收」。

三個系統互鎖：

  1. 速度：便宜累積（卡牌、尾流），易得難管。
  2. 輪胎：5 層 HP，主動衝（超加速）跟被動失誤（QTE 失敗）都會扣。
  3. 失誤牌：超車不完美時進牌庫頂污染後續抽牌；完美超車反而清淨。

回合流程（速度結算順序）v0.7

沙盒（第五關）採用五步驟回合流程。每個玩家動作（拖牌、按超車、按 PASS）都按此順序執行：

  1. 步驟 1：玩家行動（純動作 + 自身代價立即生效）
    • 從手牌移除、進棄牌堆
    • 換道 → 玩家車立即移到新道、laneCost 立即扣 playerSpeed
    • 打牌 → cardValue 立即加 playerSpeed、tireCost / penaltyNextHand 立即生效
    • 車隊牌 → 加入啟用列表（不佔動作、不走 2-4、繼續玩家輸入）
  2. 步驟 2：對手行動（過場 1 秒、無招 0.5 秒）
    • 對手評估 cooldown、出招 OR 駐守
    • 移動 → 對手車切道
    • boost → 對手 speed 永久 +amount
    • B 清道夫強招 → bypassAura = true（豁免自己光環一動）
  3. 步驟 3：檢查尾流
    • 玩家當前道 == 對手當前道？同道則 +30 playerSpeed
    • 每回合一次（超車/PASS 進新回合重置）
    • 玩家想吃尾流但對手切走 → 對手嘲諷文字（依對手個性不同）
  4. 步驟 4：賽道結算
    • 玩家結算：playerSpeed = floor((playerSpeed + add) × mult)
      • 若超過 speedLimit → 觸發彎道 QTE（玩家可化解）
    • 對手結算 v0.8 變動：
      • 對手免疫加成（add / mult / speedLimit）：順風 / 逆風 / 彎道 mult / 限速對對手都無效
      • 對手速度變化來自：
        1. 對手自己的動作：boost（+amount）、moveTo + boostAfter、stay + absBonus
        2. 賽道災害（雙方都怕、但清道夫光環免疫）：c6 油污中央道 50% 滑到鄰道、c7 坑洞速度 ÷ 2
      • 例外：若 lane bonus 有 forOpponent 子物件可顯式覆寫對手版加成（目前所有賽道未定義）
      • 詳見 對手設計
    • 切到下個賽段（影響下個動作）

超車 / PASS是平行 UI 動作、不在步驟流程內：玩家隨時可按。按超車（不同道 + 速度 > 對手）進超車 QTE；按 PASS 進防守 QTE。QTE 完之後跳過剩餘步驟、進新回合（speed/手牌重置、賽段保留）。

三個範例：

  • 打牌（標準道）：playerSpeed 50、打 +20 牌：步驟 1 → 70。步驟 4 標準道無加成 → 70。
  • 打牌（順風道，無尾流）：playerSpeed 50、順風道 +10、打 +20 牌：步驟 1 → 70。步驟 3 無同道 → 70。步驟 4 (70 + 10) × 1 → 80。
  • 換到對手道（吃尾流）：playerSpeed 60、跨 1 道、對手在目標道：步驟 1 → 60 - 10 = 50。步驟 2 對手沒切走。步驟 3 同道 +30 → 80。步驟 4 標準道 → 80。

對手在彎道（v0.8 後）：對手免疫所有賽道加成、進彎道後速度不變。舊版本對手會吃 mult 加乘、現已移除。但 c6 油污中央道（forceCornerQte）、c7 坑洞道仍會懲罰對手——加成只影響玩家，災害不分敵我。

失誤牌污染來源

失誤牌進牌庫頂，下回合必抽到。每抽到一張失誤牌就少一張可用牌——這是這版的長期壓力源。

  • 超加速 × 2（速度 +40，-1 胎）

名次規則細節

  • 初始 4 / 4，最低不能低於 4。
  • 成功超車 + 對手專注 = 0 → 名次 -1（前進）。
  • 防守失敗 → 名次 +1（後退），但 ≤ 4。
  • 每超過一名對手會把他放進 passed，可能作為後車回來追擊。
  • 名次到 1 = 通關。
