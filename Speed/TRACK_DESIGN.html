§ 0.5  賽道與賽段規格

沙盒目前已實作的賽段完整定義。玩家吃當前所在道的 add / mult / speedLimit 加成。對手從 v0.8 起免疫加成，但仍會中賽道災害（c6 油污 50% 滑、c7 坑洞速度減半）——但清道夫光環免疫災害。詳見 對手設計。

賽道類型（per-lane）

每條賽道的加成屬性。「車子在哪條賽道，車子的速度就吃那條賽道的加成」。

賽道加法加成 (add)乘法加成 (mult)限速 (speedLimit)QTE 難度 (qteDiff)
標準道01.00——
順風道+101.00——
逆風道-101.00——
內彎01.2575hard
外彎00.90105easy
急內彎01.3060hard
急外彎00.8590easy

顯示速度 = (累積速度 + add) × mult，最後無條件捨去。若顯示速度 > speedLimit → 觸發彎道 QTE（qteDiff 影響難度檔位）。

forOpponent 覆寫機制 v0.8 新

每個 lane bonus 可選擇性加上 forOpponent 子物件、單獨指定對手版的加成。若不指定（多數情況），對手對該道加成完全免疫（add=0, mult=1, speedLimit=null）。

注意：forOpponent 只控制「加成」（add / mult / speedLimit）。賽道災害旗標（如 forceCornerQte、坑洞）仍對對手生效——它們不走 forOpponent 邏輯、由 handleOpponentForceQte 跟 checkPotholeDamage 獨立判定。

// 範例 1：對手吃較弱的加成
{ lane:0, add:10, mult:1, label:"順風道 +10",
  forOpponent: { add: 5 }            // 對手只吃 +5、玩家吃 +10
}

// 範例 2：對手在彎道受速限但不吃 mult 獎勵
{ lane:0, add:0, mult:1.25, speedLimit:75, qteDiff:"hard", label:"內彎",
  forOpponent: { speedLimit: 60 }    // 對手在這道速度上限 60
}

// 不寫 forOpponent → 對手完全免疫
{ lane:1, add:-10, mult:1, label:"逆風道 -10" }

目前所有沙盒賽段都未啟用 forOpponent——對手對任何賽道加成完全免疫，但仍會中 c6 油污 / c7 坑洞災害。若 playtest 後決定恢復部分加成對對手的影響，可在 lane bonus 上加 forOpponent 子物件局部啟用。

§ 1  設計準則

新賽段要遵守的規則：

  1. 每段必須在至少一個既有維度上獨特（道數 / 彎度 / 加成類型 / 限速嚴度 / QTE 難度）。
  2. 可以引入新維度，但要在這份文件裡明確標出「需新增引擎邏輯」（避免設計完才發現接不上）。
  3. 新賽段的決策深度 ≥ c2 等級。c1 / c3 偏教學性「軟性差異」，新賽段應該都有「明顯的最佳道、但代價也明顯」的張力。
  4. 跟車牌池的互動要有想法：例如「完美過彎可換道 +50」在限速段就要重新評估、「順風 +20」在 hazard 段可能不夠用。
  5. 賽段名稱、icon、hint 要寫到位，hint 必須在一句話內讓玩家看懂「我該怎麼決策」。

c6  油污段 🛢️ 已實作 v0.7

核心張力：走中央賺 +10 加成，但要過彎道 QTE——失敗就掉胎、抽失誤牌、被滑走。整段 QTE 都變難一級。

idc6
name油污段
lanes3
bendCurve0
加成類型add（中央 +10、邊緣 -10）
QTE 難度整段 +1 級（速度檔位 +1）
lane bonuses 左道（lane 0）：add -10（外緣，安全）
中道（lane 1）：add +10（油污中央，強制彎道 QTE）
右道（lane 2）：add -10（外緣，安全）

玩家在中道（lane 1）：

  • 結算後在中道 → 強制觸發彎道 QTE（不看速度、無視 speedLimit）。
  • QTE 成功：正常吃 +10、不位移。
  • QTE 失敗：-1 輪胎 + +1 張失誤牌（沿用既有副作用）+ 滑到隨機鄰道（左道或右道、50/50）。

對手在中道（lane 1）：

  • 對手不打 QTE、改用 50% 機率失敗判定（在 resolveOpponentCircuit 之前判定）。
  • 成功（50%）：留在中道、結算用 +10。
  • 失敗（50%）：滑到隨機鄰道、結算改用該鄰道 -10（不是雙重懲罰，是把加成從 +10 變 -10）。
  • AI 期望值：對手評估中道時，期望加成 = 0.5 × (+10) + 0.5 × (鄰道平均 -10) = 0。所以對手仍會傾向中央（兩邊真實 -10 比 0 差），但不會誤把中央當純好處。

整段 QTE 難度 +1 級：

  • qteDifficultyOffset: 1 在 circuit 級別。
  • speedTierStep 內建讀取——一次覆蓋超車 QTE、彎道 QTE、節奏 QTE、防守 QTE 全部 +1 級。
  • UI 提示：每條道頂部都顯示「⚠ QTE 難度 +1 級」（橙色、所有道）。

視覺：中道（lane 1）道格遠方有紫色油污 sprite（橢圓 + 光澤 + 「油」字）。

跟既有牌互動：

  • 冷靜!（QTE 容錯 +20%）→ 在這段價值飆升。
  • 完美過彎（+50、可換道）→ 失敗被滑走後的「校正牌」。
  • 孤注一擲（+60，下回合手牌 -1）→ 走中央 +10 加 +60 是高速峰值，但 QTE 失敗代價也更重。

c7  坑洞段 🕳️ 已實作 v0.7

核心張力：3 道 add 都 +5，但其中 2 道有坑、1 道安全。坑洞位置玩家可見、對手會自動避——但對手沒 action 那回合避不了。

QTE 難度普通

坑洞規則：

  • 進入 c7 時（applyCircuit）隨機抽 0-2 之一為安全道、其他兩道為坑洞道。
  • 坑洞道在 UI 上有明顯 sprite（黑橢圓 + 紅外框 + 「坑」字）—— 玩家可見。
  • 結算後人在坑洞道：
    • 玩家：-1 輪胎。
    • 對手：速度 / 2（對手沒胎系統，用降速代替）。
  • 切換到下個賽段時清空 potholeLanes。

對手 AI 避坑（決策層）：

  • 對手選道時自然排除坑洞道（在 pickSmartLaneForOpponent、moveTo、moveAdjacent 各處）。
  • 但只有對手「這回合有 action」才會避——idle（cooldown 沒到）的回合不能換道、原本在坑就吃坑（speed /2）。
  • 沒有「結算前最後一刻瞬移」的不合理感——避坑發生在 AI 決策時。
  • 例外：清道夫不需要避坑（光環免疫）、AI 不對清道夫做避坑處理。

清道夫(B)光環互動：

  • 清道夫光環免疫所有賽道災害：當前對手是清道夫時、踩到任何坑洞道 → 不降速；踩到油污中道 → 不會 50% 滑開。
  • 玩家在「清道夫所在道」踩坑 → 也免疫（光環延伸到玩家）。
  • 玩家可以主動跟到清道夫所在道避坑——多了一個戰術層。

視覺：坑洞道頂部畫黑橢圓 sprite + 紅色警告外框 + 「坑」字。

駛過揭曉機制：

  • 對手駛過某道不揭曉——只玩家走才揭。

  • 下一賽段預告：所有道全顯示「紅道: ?」等，徹底不洩漏。
